How do you manage the code base of a software project?

  • Local file system
  • Central Server (VCS)
  • Distributed (DVCS)

Here comes Git!

Next generation VCS, it's distributed so rather than one place for the full version history of a codebase, each developer's working copy contains the full project history.

What is git?

British slang for "pig headed, think they are always correct, argumentative"

Developed by Junio Hamano and Linus Torvalds in 2005.

Built specifically to manage the Linux kernel.

Git has been designed with performance, security and flexibility in mind.

Perfomance

Changes made to local file system, no network connection needed.

Uses a combination of delta encoding (storing content differences), compression and explicitly stores directory contents and version metadata objects.

Security

Git repository are secured with a secure hashing algorithm, SHA1.

Flexability

Supports multiple development workflows.

Works well with project large and small.

Not for just code. Can be used for any text files.

How to git

Install git

What is a git repo?

A git repo is broken down into three "trees".

  • Working directory: Your local file system.
  • Index: A virtual directory which stages files that are ready for commit.
  • History: Record of every commit made to the project.

Head: Pointer to your last commit. (Top of the tree)

Creating your repo

$ git init

$ git clone http://path/to/remote.git .

Starting with the basics

$ git add [file/folder]
$ git commit -m "Inital commit"

Sharing your changes

git push orign master

What is origin?

Alias of server info.

Getting contributed changes.

$ git pull

Tagging

Tagging allows you to mark releases or milestones in your project.

$ git tag [version] [commit id]

Branching and Merging

Branching allows you to work on features and code changes independently from you core base.

The default main branch is called master.

Changes you want to keep can then be merged into master.

Branching

$ git checkout -b newFeature

Merging

$ git merge newFeature

How do I go back?

If you have local changes that you want to completely revert.

$ git checkout [filename]

If you have staged changes that you want to reset.

$ git reset [filename]

How do we roll back?

Checkout all files from a old commit.

$ git checkout [commit] [filename]

Checkout a previous version of a file.

$ git checkout [commit] [filename]

My repo is totally housed!

$ rm -rf .git
$ git init

Git Logs

A git repo contains the history of all the changes to a project.

$ git log
$ git log --graph --oneline --decorate --all

Git Flow

A strict branching model designed around the project release. It provides a robust framework for managing larger projects.
